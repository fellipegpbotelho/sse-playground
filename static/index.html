SSE Wire Format — Quick Reference

FieldExamplePurpose
data:data: {"msg":"hi"}The payload. Multi-line data uses multiple data: lines.
event:event: alertNamed event type. Browser calls matching addEventListener('alert', …).
id:id: 42Event ID. Browser stores as lastEventId and sends as Last-Event-ID header on reconnect.
retry:retry: 3000Override browser reconnect delay (ms). Default is ~3000 ms.
: comment: heartbeatIgnored by browser, but resets proxy timeout clocks.

Events are separated by a blank line (\n\n). The content-type must be text/event-stream with Cache-Control: no-cache.
